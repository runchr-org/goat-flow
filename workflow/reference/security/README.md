# Security Reference Library

Reference material for later local-instruction or audit work.

These files are not base setup steps and should not be copied verbatim into projects. Use them only after reading the real codebase and project rules.

Available topics:

- auth and API boundaries
- file upload handling
- infrastructure and deployment risks
- LLM integration risks
- PHI / compliance concerns
- secrets management
- SQL injection
- supply-chain security
- common web risks
- framework-specific notes for Django, Express/Node, Go, Laravel, and Symfony
