# Template: Mob Elaboration

> This is a reference template. Not installed to projects by default.

Use this template for second-pass requirements sharpening after a feature brief.

For each round, ask exactly 3 to 5 clarifying questions and check:

1. Business rules and hard constraints
2. Edge cases and failure modes
3. Integration points with existing systems
4. User-visible outcomes and acceptance criteria
5. Non-goals and blast-radius limits

### Question Quality

- Ask about actual behavior-changing assumptions first
- Ground questions in context when available
- Present forks clearly when multiple interpretations are possible

### Exit and locking

- Stop when there are two consecutive rounds with no meaningful new scope changes
- Stop when user answers become only confirmatory
- Stop when all HIGH-priority ambiguities from round 1 are answered
- Hard cap: no more than 4 rounds without explicit consent to continue

### When Locked

Produce a requirements summary with:

- Locked requirements
- Non-goals
- Constraints
- Failure modes / edge cases
- Integration notes
- Open decisions
