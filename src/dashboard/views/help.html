Getting Started
What is GOAT Flow?

AI coding agents skip verification steps, create duplicate files instead of editing in place, and repeat the same mistakes across sessions. GOAT Flow gives them structure: an audit that validates their setup, hooks that block dangerous commands, and a learning loop that captures lessons so mistakes don't recur. Think of it as guardrails and memory for your AI agent.

How the Audit Works

The auditor evaluates your agent setup across three scopes, all pass/fail:

GOAT Flow Setup
Required files, config, skills, instruction files, preamble, and conventions.
Agent Setup
Per-agent checks: instruction file, skills, settings, deny mechanism.
AI Harness Completeness
With --harness: deterministic integrity checks across 5 concerns (context, constraints, verification, recovery, feedback loop).

All checks are deterministic pass/fail. Use critique for agent-driven quality assessment.

What Are Skills?

Skills are slash commands that give your AI agent structured workflows instead of free-form prompting.

/goat-debug Diagnose bugs, explore code, investigate unfamiliar areas
/goat-plan Plan features, refactors, and milestones with complexity routing
/goat-review Review diffs, audit code quality with negative verification
/goat-security Threat model, dependency audit, compliance checks
/goat-test Generate test plans with automated, AI-verified, and manual steps
/goat Dispatcher that classifies your intent and routes to the right skill
What Are Hooks?

Hooks are safety nets that run automatically before or after agent actions.

deny-dangerous Blocks destructive commands like rm -rf, force push, and secret exposure
Projects can add their own hooks (e.g. lint validation) alongside deny-dangerous.
What Is the Learning Loop?

Agents forget everything between sessions. The learning loop gives them persistent memory through four types of records:

Footguns
Architectural traps with file:line evidence. Prevents the agent from hitting the same code landmine twice.
Lessons
Behavioral mistakes the agent made. Logged so the same error pattern is recognized and avoided.
Decisions
Architecture Decision Records. Captures why a choice was made so future agents don't reverse it.
Session Logs
End-of-session summaries. Provides continuity between work sessions.
What Is the Execution Loop?

Every agent action follows a four-step loop that prevents the most common failure modes:

READ
Read relevant files before making changes. Never guess.
SCOPE
Declare which files will change and what won't.
ACT
Make changes within the declared scope.
VERIFY
Run linters, check cross-references, confirm correctness.
