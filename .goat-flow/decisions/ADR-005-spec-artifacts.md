# ADR-005: goat-plan produces structured spec artifacts

**Date:** 2026-03-22
**Status:** Superseded — entire workflow retired in v1.1.0. `requirements-{feature}.md`, `REQ-NNN` IDs, the conditional Phase 0 in goat-review, and `workflow/playbooks/planning/requirements-template.md` have all been removed. Current goat-plan uses milestone files with assumptions + exit criteria; current goat-review uses the opt-in Spec Drift check against milestone exit criteria and assumptions. See `.claude/skills/goat-plan/SKILL.md` and `.claude/skills/goat-review/SKILL.md` for the replacement flow.

## Context (historical)

goat-review had a conditional Phase 0 that checked for `requirements-{feature}.md` and validated diffs against its acceptance tests. No skill was responsible for producing the structured format that Phase 0 expected. The requirements files generated by goat-plan Phase 1 were narrative-only -- they lacked parseable requirement IDs and binary acceptance tests.

Without a structured format, goat-review's spec compliance check had nothing machine-readable to verify against.

## Decision (historical — superseded)

goat-plan Phase 1 produced a structured requirements section inside `requirements-{feature}.md` for Standard complexity or higher. The format used:

- Sequential `REQ-NNN` IDs for each requirement
- A **Description** and binary **Acceptance Test** per requirement
- Per-requirement **Status** tracking (`pending | pass | fail`)
- A **Non-Requirements** section for explicitly excluded scope
- A **Verification Summary** with counts

The template lived at `workflow/playbooks/planning/requirements-template.md` (removed). The structured section was appended after narrative brief sections. Hotfix-complexity features skipped it.

## Why superseded

`workflow/playbooks/` was retired in v1.1.0 along with the `REQ-NNN` machinery. The replacement surfaces are:

- goat-plan milestone files: `Assumptions`, `Exit Criteria`, `Testing Gate` (per milestone).
- goat-review Spec Drift (opt-in): reads active milestone's exit criteria and assumptions, emits `[advisory]`, `[ready-to-tick]`, or `[MUST:needs-decision]` findings.
- No more shared file format between goat-plan and goat-review — the coupling is via the milestone file in `.goat-flow/tasks/<active>/`.
