## 6 — SBAO-2 round on uncritiqued infrastructure batch (M13/M14/M16/M18)

After the main restructure and M19 rewrite, a fresh-context-readiness review of `1.2.0/` surfaced two remaining gaps: (1) the glossary didn't define several terms the milestone files use (`CSO`, `rationalization table`, `harness`, `concern`, `integrity/advisory/metric`, `rubber-stamp`), and (2) four milestones had been flagged in the prior SBAO's "What Wasn't Critiqued" section and never received sub-agent scrutiny. Ran a second full-mode SBAO round specifically on M13, M14, M16, M18.

Three sub-agents returned convergent findings on four topics:

- **M14 schema is too weak.** `{ pattern, glob, expect: "present"|"absent" }` cannot express constraints the milestone's own Objective cited ("all API routes have auth middleware" requires AST, not grep). Strong consensus A+B+C. Rescoped to `{ id, name, command, expect_exit, timeout_seconds? }` per Agent B's alternative — shell-command primitive lets users chain `rg`/`eslint`/custom scripts. Default 30s timeout with per-constraint override. Consistent with existing hooks/scripts trust model.
- **M16 is overbuilt.** Agent B's alternative (replace ~200 lines of hashed-sentinel infrastructure with a ~25-line quarterly checklist workflow) was cross-examined against actual repo drift history. Verification: 2 Codex-spec drift incidents in the last 2 weeks (`commits 1e54a76`, `2ea08c1`; resolved footguns `hooks.md:37-38`), both caught by existing audit/critique process. M16's own prior kill criterion said "<1 drift/quarter → simplify to quarterly manual review" — we're starting where the kill criterion pointed. Upgrade trigger: if 2 consecutive quarters surface real drift in the same URL, add a hashed sentinel for that URL only.
- **M18 §3 (prompts tier) dropped.** Pattern-matching rendered prompts for "required sections" / "banned terms" would fail on legitimate composer rewrites — the exact flaky-test failure M18's own kill criterion warns about. B's alternative (assert composer return type instead of rendered strings) requires refactoring the composers to return structured data first; out of scope. `compose-setup.ts` and `compose-critique.ts` can have unit tests separately if needed. M18 retains hooks tier (§1) and skills tier (§2) which are tied to real incidents.
- **M18 time budgets fixed.** Agent C (fresh eyes) caught that per-tier budgets (10s + 20s + 30s = 60s) summed to the "combined M13+M18 ≤60s" cap before M13 added anything — broken math. Revised to ≤20s per tier + combined M13+M18 ≤90s.

Other adopted changes:
- **M13** rescoped from 3 language fixtures (TypeScript / Python / pre-existing-docs) to 1 minimal TS fixture × 3 setup scenarios (fresh / existing `.goat-flow/` / existing `CLAUDE.md`), because setup writes to `.goat-flow/` and `ai-docs/` regardless of host language (per `feedback_setup_file_ownership.md`). Language axis is orthogonal; scenario axis is the one that matters.

- **M14 "agent-behaviour note"** deleted per Agent B — it was hypothetical counter-rationalization for a behaviour nobody had observed, violating CLAUDE.md Hard Rule "MUST use real incidents, never hypothetical."
- **M08 `--skip-prompts` exit criterion** strengthened to an explicit cross-milestone contract, closing Agent A's finding that M13 depended on a flag name M08 didn't lock down.
- **Glossary extended** with 6 missing terms: CSO, rationalization table, harness, concern, integrity/advisory/metric, rubber-stamp. Closes the fresh-context-readiness gap from C1 (jargon).
